下面为你详细介绍 Redis 中几种核心数据类型的定义、特性、常用命令与典型应用场景。

### 📊 各数据类型对比

| 数据类型 | 特性 | 核心命令 | 应用场景 |
| :--- | :--- | :--- | :--- |
| **String (字符串)** | 二进制安全，最大512MB | `SET`、`GET`、`INCR`、`MSET` | 缓存、计数器、分布式锁、会话存储 |
| **List (列表)** | 有序、可重复，基于链表实现 | `LPUSH`、`RPUSH`、`LPOP`、`RPOP`、`LRANGE` | 消息队列、时间线、最新动态列表 |
| **Set (集合)** | 无序、元素唯一，自动去重 | `SADD`、`SREM`、`SISMEMBER`、`SMEMBERS` | 标签系统、共同好友、唯一性判断 |
| **ZSet (有序集合)** | 有序、元素唯一，关联分数排序 | `ZADD`、`ZRANGE`、`ZRANK`、`ZINCRBY` | 排行榜、优先级队列、延时队列 |
| **Hash (哈希)** | 键值对集合，类似小字典 | `HSET`、`HGET`、`HGETALL`、`HDEL` | 存储对象、购物车、计数器分组 |
| **Stream (流)** | 消息队列结构，支持持久化和ACK | `XADD`、`XREAD`、`XREADGROUP` | 消息队列、事件溯源、实时数据管道 |
| **Vector Set (向量集)** | 存储和检索向量，支持相似度搜索 | `VADD`、`VSIM` | 语义搜索、推荐系统、AI应用 |

### 💎 核心数据类型详解

#### String (字符串)
String 是 Redis 中最基础的数据类型，一个 key 对应一个 value。它是**二进制安全**的，可以存储任何形式的数据，例如字符串、整数、JSON、甚至是序列化的图片，最大容量为 512MB。

#### List (列表)
List 是简单的字符串列表，**按照插入顺序排序**。它的底层是**双向链表**，因此对列表两端（头/尾）的插入和删除操作非常快，时间复杂度为 O(1)。

#### Set (集合)
Set 是 String 类型的**无序集合**，集合中的元素是**唯一且不重复**的。得益于其哈希表实现，Set 的添加、删除、查找操作的时间复杂度都是 O(1)。

#### ZSet (有序集合)
ZSet 是 Set 的升级版，它为每个元素关联一个**分数（score）**，并依据分数进行自动排序。它的元素仍需要唯一，但分数可以重复。

#### Hash (哈希)
Hash 是键值对（field-value pair）的集合，特别适合存储对象。相比用多个 String 来存储一个对象，Hash 能更高效地进行单字段的读写和内存管理，同时保持操作原子性。

#### Stream (流)
Stream 是 Redis 5.0 引入的专门用于处理**消息队列和事件流**的数据类型。它克服了传统 List 作为队列时的一些不足。

#### Vector Set (向量集)
Vector Set 是较新的数据类型，专为**向量相似度搜索**设计。它为每个元素关联一个多维浮点数向量，并通过 HNSW 等算法实现高效的相似度查找。

### 💎 总结与选择建议

选择合适的数据类型对高效利用 Redis 至关重要。

*   **简单存储和计数**：选 **String**，最基础、性能最高。
*   **有序集合与去重需求**：需要**元素不重复**但**无需排序**，选 **Set**；需要**元素不重复**且**按权重排序**（如排行榜），选 **ZSet**。
*   **存储对象与部分更新**：如果应用频繁更新对象的某个字段，选 **Hash**；如果总是整体读写，用 String 存序列化后的 JSON 也可以。
*   **消息队列与流处理**：简单的**生产者-消费者模型**，用 **List** 即可；需要**消费者组**、**消息确认**（ACK）和**消息回溯**等高级功能，选 **Stream**。
*   **AI与相似度搜索**：构建语义搜索、推荐系统，选 **Vector Set**。
